## Step 1：HTML 骨架 + CSS 设计系统

**目标**：搭建完整页面结构和全局样式变量

**输入**：
- design-spec.md（色彩、字体、间距）
- requirements.md（功能需求）

**详细任务**：
1. 创建 `index.html`，包含所有区域的语义化 HTML 骨架
2. 创建 `styles.css`，定义：
   - CSS Variables（颜色、字体、间距、圆角、阴影）
   - Reset / Normalize 基础样式
   - 导航栏样式（固定顶部、毛玻璃背景）
   - 全局排版样式（标题、正文、链接）
3. 创建 `script.js` 基础框架（导航栏滚动效果、回到顶部按钮）

**产出物**：
- `index.html`（完整骨架）
- `styles.css`（设计系统变量 + 基础样式）
- `script.js`（基础交互框架）

**验证标准**：
- 浏览器打开 `index.html`，页面无报错
- 各区域可见（内容为占位或已填充）
- CSS 变量通过浏览器 DevTools 检查正确
- 导航栏固定在顶部

## Step 2：Hero 个人信息区

**目标**：填充 Hero 区的真实内容

**详细任务**：
1. 样式化照片容器（圆形裁剪、发光边框）
2. 排版姓名 + 岗位标题
3. 能力标签（药丸形徽章）
4. 个人简介段落
5. 3 条核心优势（图标 + 文字）
6. 联系方式（电话/邮箱/学历图标列表）

**验证标准**：
- 照片正确显示、圆形裁剪
- 所有文字内容与 PDF 简历一致
- 标签排列美观
- 向下滚动提示动画正常

---

## Step 3：时间轴组件

**目标**：实现教育/工作经历时间轴

**详细任务**：
1. CSS 时间轴布局（中间竖线 + 左右分支）
2. 时间轴卡片样式
3. 节点圆点 + 发光动画
4. 填充真实数据（2条教育 + 2条工作）
5. 滚动入场动画（Intersection Observer）

**验证标准**：
- 桌面端左右双栏清晰可辨
- 4 张卡片时间、内容正确
- 节点发光动画流畅
- 滚动到时间轴区域时卡片依次入场

---

## Step 4：项目卡片 + 弹窗

**目标**：实现项目卡片和详情的完整交互

**详细任务**：
1. 卡片正面：渐变背景图（模拟）+ 图标 + 标题 + 概述 + 标签
2. 卡片 hover 动效（上浮 + 发光增强）
3. 弹窗（Modal）组件：
   - 打开/关闭动画
   - 点击遮罩关闭
   - Esc 键关闭
   - 滚动锁定（弹窗打开时禁止背景滚动）
4. 弹窗内容：脱敏后的项目详情（结构化呈现）

**验证标准**：
- 2 张项目卡片正确渲染
- 点击卡片弹出详情弹窗
- 弹窗内项目内容完整、脱敏
- 关闭弹窗（3 种方式均可）
- 移动端弹窗全屏显示

---

## Step 5：专题研究区

**目标**：在项目卡片下方添加专题研究区

**详细任务**：
1. 复用项目卡片组件样式
2. 专题研究使用不同标签颜色（强调色区分）
3. 填充 2 个专题研究的弹窗内容：
   - 轨交研究：聚焦巴黎地铁 RATP + IDFM 模式
   - 资源循环研究：5 种路径 + 5 个案例

**验证标准**：
- 2 张研究卡片正确渲染
- 弹窗内容准确、结构清晰
- 与项目卡片视觉区分明确

## Step 6：下载区 + 导航

**目标**：完成下载区和导航栏交互

**详细任务**：
1. 下载区样式（居中布局、大按钮）
2. PDF 下载链接正确配置
3. 导航栏平滑滚动（点击导航项滚动到对应区域）
4. 导航栏滚动后背景显现
5. 移动端汉堡菜单展开/收起

**验证标准**：
- 点击下载按钮触发 PDF 下载
- 导航项点击后平滑滚动到对应区域
- 滚动页面后导航栏出现毛玻璃背景
- 移动端菜单正常工作

## Step 7：响应式 + 动效打磨

**目标**：全站响应式适配和细节动画优化

**详细任务**：
1. 平板断点适配（768-1024px）
2. 手机断点适配（<768px）
   - 时间轴切换单列
   - 卡片全宽
   - Hero 区改为上下布局
   - 导航折叠为汉堡菜单
3. 背景粒子动画
4. 各区域入场动画微调
5. 性能优化（减少重绘）

**验证标准**：
- Chrome DevTools 模拟 iPhone/Pad 正常显示
- 所有文字可读、按钮可点击
- 无横向滚动条
- 动画流畅无卡顿

## Step 8：GitHub Pages 部署

**目标**：网站上线可公开访问

**详细任务**：
1. 创建 `.gitignore`
2. 初始化 Git 仓库
3. 创建 GitHub 仓库
4. 推送代码
5. 开启 GitHub Pages
6. 验证线上访问

**验证标准**：
- 手机和电脑均可访问线上 URL
- 所有功能正常
- PDF 下载链接正常
